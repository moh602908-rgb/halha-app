# دلّني AI

**الإصدار الحالي: v1.2.0**

حلول عملية للحياة اليومية — تطبيق ويب (PWA) يقدّم أدلة خطوة بخطوة لحل أكثر مشاكل الحياة اليومية شيوعًا، بالعربية، ويعمل بالكامل دون إنترنت بعد أول زيارة، مع بحث ذكي يفهم الأخطاء الإملائية والمرادفات.

## نطاق هذا الإصدار (Version 1.0)
- **24 مجالًا متوازنًا**، كل مجال يحتوي **7 أدلة كاملة بالضبط** (168 دليلاً إجمالاً)، بدون أي دليل "قيد الكتابة" وبدون أي عنوان مكرر.
- تمت مراجعة كل المحتوى للتأكد من خلوّه من أي استشارة طبية أو قانونية أو نفسية متخصصة، أو محتوى حساس (حمل/ولادة، علاقات زوجية/حميمية)، ليتوافق مع سياسات متاجر التطبيقات مثل Google Play.
- **بحث محلي ذكي بالكامل** (بدون إنترنت أو أي خدمة خارجية): يتجاهل الهمزات/التاء المربوطة/الألف المقصورة/التشكيل، يتسامح مع خطأ إملائي بسيط، يفهم مرادفات شائعة (جوال/هاتف، فلوس/مال...)، ويعرض أقرب النتائج بدل "لا توجد نتائج" عند عدم وجود تطابق تام.
- نافذة **"مساعد دلّني"**: وصول سريع لأكثر الأدلة استخدامًا + نفس محرك البحث الذكي، بدون ذكاء اصطناعي في هذه المرحلة (سيُضاف في المرحلة الثانية عبر خادم خاص، وليس مفتاح API مكشوف في المتصفح).
- يعمل كتطبيق PWA قابل للتثبيت على أندرويد وiOS والحاسوب، بأيقونات PNG قياسية (192/512 + نسخة maskable)، ودعم تصفح دون إنترنت بعد أول زيارة.

## سجل مراجعة v1.2.0 (آخر مراجعة قبل الاعتماد)
- **إصلاح جذري**: مربع البحث الرئيسي كان يفقد التركيز (Focus) بعد كل حرف، لأن الصفحة كانت تعيد إنشاء عنصر `<input>` بالكامل مع كل نتيجة بحث. أُصلح عبر فصل حقل البحث عن منطقة النتائج بحيث لا يُعاد إنشاء الحقل أبدًا أثناء الكتابة.
- **بحث ذكي جديد بالكامل**: تطبيع الكتابة العربية (أ/إ/آ ← ا، ة ← ه، ى ← ي، إزالة التشكيل)، مرادفات شائعة، بحث تقريبي يتسامح مع خطأ إملائي، وعرض "أقرب النتائج" بدل رسالة فارغة. يعمل بالكامل محليًا دون إنترنت، وسرعته دون الميلي ثانية الواحدة حتى مع 168 دليلاً.
- إضافة أيقونات PNG قياسية (192 / 512 / نسخة maskable / نسخة iOS) لضمان أفضل توافق تثبيت على كل الأجهزة.
- إصلاح تكرار أيقونة 🌱 بين مجالين (أصبحت الزراعة 🪴).
- مراجعة شاملة أكدت: 24 مجالًا × 7 أدلة بالضبط = 168، بلا تكرار، بلا محتوى حساس، الكل بنفس الأسلوب.
- اختُبر التطبيق فعليًا بمتصفح Chromium حقيقي (ليس تخمينًا): البحث، التنقل، نافذة المساعد، العمل دون إنترنت، ودورة نشر تحديث كاملة — كل الاختبارات نجحت.

## الملفات
- `index.html` — الصفحة الرئيسية وهيكل التطبيق
- `style.css` — التصميم والألوان
- `app.js` — بيانات الأدلة (24 مجالًا متوازنًا)، محرك البحث الذكي، ومنطق التنقل ونافذة المساعد
- `manifest.json` — إعدادات PWA (قابلية التثبيت)
- `icon.svg` — الأيقونة الأساسية (متجهية، تُستخدم كخيار احتياطي عالي الجودة)
- `icon-180.png` — أيقونة شاشة الهاتف الرئيسية لأجهزة iOS تحديدًا (Apple Touch Icon)
- `icon-192.png` / `icon-512.png` — أيقونات PWA القياسية لأندرويد والحاسوب
- `icon-512-maskable.png` — نسخة مخصصة لأندرويد بهامش أمان كافٍ حتى لا تُقص عند تطبيق قصّ الشكل التلقائي للأيقونات (Adaptive Icons)
- `icon-maskable.svg` — الملف المصدر (متجهي) لتوليد `icon-512-maskable.png`؛ غير مستخدم مباشرة وقت تشغيل التطبيق، محفوظ فقط لتسهيل تعديل الأيقونة مستقبلاً
- `sw.js` — العمل دون إنترنت + التحكم بتحديث النسخة المعروضة للمستخدم
- `vercel.json` — يمنع تخزين الملفات مؤقتًا على مستوى الاستضافة/الشبكة، فيضمن وصول التحديثات فورًا

## ⚠️ تنبيه مهم جدًا لأي تحديث مستقبلي
في كل مرة تُحدَّث فيها ملفات المشروع ثم تُرفع من جديد، **يجب تغيير رقم `APP_VERSION` في أعلى ملف `sw.js`** (مثلاً من `v1.2.0` إلى `v1.3.0`).
بدون هذه الخطوة، سيستمر متصفح كل مستخدم بعرض **النسخة القديمة المحفوظة عنده**، حتى لو نجح الرفع على GitHub وVercel بنجاح تام.

## 🐛 إصلاح مشكلة "النسخة القديمة تظهر رغم الرفع" (تمّ حلها في v1.1.0)
السبب الجذري الذي عُثر عليه: داخل `sw.js`، كان استدعاء `fetch(req)` بدون تحديد وضع الكاش يستخدم الوضع الافتراضي، والذي يسمح للمتصفح بالإجابة من ذاكرته الداخلية (HTTP Cache) دون الاتصال الفعلي بالشبكة — رغم أن نية الكود كانت "الشبكة أولًا دائمًا". هذا كان يُبطل التحديث حتى مع تغيير `APP_VERSION`، وأحيانًا حتى في التصفح المتخفي إن كانت هناك طبقة تخزين مؤقت وسيطة (كخادم Vercel نفسه) تُرجع نسخة قديمة.
تم إصلاح هذا نهائيًا عبر:
1. **`sw.js`**: كل الطلبات المارة عبر الـ Service Worker تستخدم الآن `{ cache: "no-store" }` صراحة، فتُجبر على الاتصال الفعلي بالخادم دائمًا (لا اعتماد على أي ذاكرة HTTP وسيطة).
2. **`index.html`**: تسجيل الـ Service Worker يستخدم الآن `updateViaCache: "none"` (يمنع تخزين ملف `sw.js` نفسه مؤقتًا)، ويطلب فحص تحديث فوري (`reg.update()`) عند كل تحميل صفحة، ويعيد تحميل الصفحة تلقائيًا مرة واحدة فور تفعيل نسخة جديدة — فلا يحتاج المستخدم لتحديث يدوي أو مسح بيانات الموقع أبدًا.
3. **`vercel.json`** (ملف جديد): يضبط رؤوس `Cache-Control: no-cache, no-store, must-revalidate` على كل الملفات عند النشر على Vercel، لمنع أي تخزين مؤقت على مستوى الخادم أو الشبكة (CDN) نفسه. العمل دون إنترنت لا يتأثر بهذا إطلاقًا، لأنه يعتمد بالكامل على `Cache Storage` الخاصة بالـ Service Worker، وهي منفصلة تمامًا عن ذاكرة HTTP Cache العادية للمتصفح.

**ملاحظة لمرة واحدة فقط:** المستخدمون الذين زاروا الموقع قبل هذا الإصلاح لديهم نسخة قديمة من `sw.js` نفسه ما تزال تعمل مؤقتًا في متصفحهم. زيارتهم التالية للموقع (حتى دون أي إجراء يدوي منهم) ستكتشف تلقائيًا وجود `sw.js` جديد، وتحدّث نفسها، وتُعيد تحميل الصفحة تلقائيًا مرة واحدة — دون أي تدخل من المستخدم.

## لا يحتاج أي تثبيت
هذا مشروع HTML/CSS/JavaScript خام بدون أي أدوات بناء (لا Node.js ولا npm مطلوبة لتشغيله أو نشره). يكفي رفع الملفات كما هي إلى الاستضافة.

## سياسة المحتوى (مهم عند إضافة أدلة جديدة لاحقًا)
لتفادي أي مشكلة مستقبلية مع متاجر التطبيقات، تجنّب دائمًا أي دليل جديد يتضمن:
- تشخيصًا طبيًا أو وصف جرعات/علاجات دوائية محددة.
- محتوى عن الحمل والولادة والرعاية الطبية للمولود.
- محتوى عن العلاقات الزوجية أو الحميمية أو الصحة الجنسية.
- استشارة قانونية أو نفسية متخصصة (تحويل المستخدم لمختص عند الحاجة مقبول، لكن ليس تقديم الاستشارة نفسها).
الأدلة الحالية كلها إما مهارات عملية عامة، أو معلومات سلامة وإسعافات أولية أساسية (لا تُعتبر استشارة طبية متخصصة)، أو توعية عامة مع توجيه واضح لمختص عند الحاجة.

## إضافة دليل أو مجال جديد لاحقًا
أضف عنصرًا جديدًا داخل مصفوفة `guides` الخاصة بالمجال المناسب في `app.js`، بنفس شكل الأمثلة الجاهزة (`title` + `ready: true` + `steps` + `note` اختياري). حافظ على 7 أدلة لكل مجال للحفاظ على التوازن. لإضافة مجال جديد بالكامل، أضف كائنًا جديدًا لمصفوفة `DOMAINS` بنفس البنية (`id` فريد، `name`، `icon`، `guides`) بـ7 أدلة كاملة منذ البداية.

## المرحلة الثانية (لاحقًا)
دمج مساعد ذكي فعلي عبر خادم وسيط خاص (بدل استدعاء API من المتصفح مباشرة بمفتاح مكشوف كما كان مخططًا سابقًا)، مع إبقاء نافذة الوصول السريع الحالية كخيار يعمل دائمًا دون إنترنت.
